## git 常用命令和基本原理

###### git log

查看历史纪录的命令，如果需要显示的简洁使用--oneline参数，如果需要图形化的界面可以使用--graph参数

###### git diff

这条命令可以查看执行git status后的详细信息，可以查看写入缓存和已经修改但是未写入缓存的改动区别。直接git diff是查看尚未缓存的改动， 如果加上--cached参数则查看已经缓存过的改动， --HEAD参数则指某个已缓存和未缓存的文件的区别。

###### git rm

删除命令，和git add命令应该是相反的一个命令。

命令的使用方式为git rm filename

其中会存在使用某个参数，比如如果想彻底在跟踪的文件清单中删除某个文件就使用 git rm -f命令

如果只是想从暂存区删除，但是本工作目录中还是保留，则使用git rm --cache命令

###### git mv

从命令的名字来看应该是移动某个文件，但是如果是显式地调用这个命令的话，一般是用来进行文件的重命名。

命令的使用方式很简单，就是 git mv oldfilename newfilename

###### git reset

git reset 的作用是修改HEAD指针的位置，就是将HEAD指向的位置改变为之前存在的某个版本。用这个命令的前提是，你想回退到某个之前的版本，并且保证一点的是，你后面的一些版本也不需要了

命令格式 git reset --<option>  HEAD

HEAD的意思个人理解就是指某个版本的版本号。那么关于option其实有三种不同的参数

--soft  改变HEAD指针，但是索引和工作区不变

--mixed 改变HEAD指针和索引内容，但是工作区不变

--hard 改变三者，也就是说会造成给定提交后的修改内容的丢失。
